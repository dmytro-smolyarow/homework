# Spec: global invariants

Declarative structural rules that hold for **every** Cloudflare Workers + Hono service built with this skill. After any change, self-verify against this list. Each rule is a `MUST` / `MUST NOT` with a **Check** hint (grep pattern or visual cue) â€” no script to run, the model reads and confirms.

## Barrels

- **MUST** ship an `index.ts` at every slice and segment folder.
  Check: each new `<slice>/` or `<segment>/` folder has `index.ts`.
- **MUST NOT** place a barrel at layer level (`modules/`, `widgets/`, `features/`, `entities/`, `shared/`, `pkg/`).
  Check: `ls src/app/modules/index.ts` (and siblings) â†’ does not exist.

## Import direction

- **MUST** import only downward: `routes â†’ modules â†’ widgets â†’ features â†’ entities â†’ shared`.
- **MUST NOT** import upward or sideways within a layer (moduleâ†’module, featureâ†’feature, widgetâ†’widget).
  Check: `grep -R "from '../../modules" src/app/modules/` returns nothing.

## Routing boundary

- **MUST** keep Hono routing in `modules`, the `routes` aggregator, and `<entry>.ts` only.
- **MUST NOT** call `app.route(...)` / define routes inside `widgets`, `features`, `entities`, or `shared`.
  Check: `grep -R "app.route\|new Hono" src/app/{widgets,features,entities,shared}` returns nothing.

## Durable Object placement

- **MUST** define every Durable Object class in `entities/models/<entity>.object.ts`.
- **MUST** re-export each DO class from its owning `app/<entry>.ts`.
  Check: `grep -R "extends DurableObject" src/app` only matches `entities/models/*.object.ts`; the entry re-exports the class.

## `pkg/` self-containment

- **MUST NOT** let a `pkg/*` slot import from `app/*` or from another `pkg/*`.
  Check: `grep -R "from '@/app/\|from '../app" src/pkg/` returns nothing.
- **MUST** validate routes through `pkg/validator/` (the `zValidator` wrapper), never `@hono/zod-validator` directly.
  Check: `grep -R "@hono/zod-validator" src/app` returns nothing.

## Service / handler shapes

- **MUST** keep `<module>.module.ts` thin â€” validate + delegate only; domain logic lives in `<module>.service.ts`.
- **MUST** return `ctx.json(...)` on every service path, including the standard 500 in `try/catch`.
- **MUST** schedule cache invalidation via `ctx.executionCtx.waitUntil(...)` on every mutation (`POST`/`PATCH`/`DELETE`).
  Check: each mutation route calls `waitUntil` to invalidate affected cache keys.

## Wrangler & generated types

- **MUST** mirror any binding / DO / migration / var across **all** `wrangler.<entry>.jsonc` env blocks (no inheritance).
- **MUST NOT** hand-edit `types-<entry>.d.ts` (generated by `wrangler types` / `yarn cf-typegen`).
- **MUST NOT** commit secret values into `wrangler.<entry>.jsonc` (use `wrangler secret put`).

## Environment access

- **MUST** read env only through `envConfig` from `config/env.config.ts`.
- **MUST NOT** read `process.env` outside `config/env.config.ts`.
  Check: `grep -R "process.env" src --include=*.ts | grep -v "config/env.config.ts"` returns nothing.

## File & folder naming

- **MUST** carry the role suffix on every implementation file (`.module.ts`, `.service.ts`, `.object.ts`, `.table.ts`, `.collection.ts`, `.model.ts`, `.dto.ts`, `.interface.ts`, `.constant.ts`, `.middleware.ts`, `.util.ts`, `.config.ts`, `.pkg.ts`, `.routes.ts`).
- **MUST** use kebab-case folders; the slice folder name equals the file prefix.

## Layer purity

- **MUST** keep `shared/utils/*.util.ts` pure (no Hono, no I/O, no service calls).
- **MUST** keep `entities/models` and `entities/dto` from importing each other.
